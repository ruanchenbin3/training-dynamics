❌ 关键问题： Figure 2 的 1D 插值图 没有做任何归一化。 大 batch 的权重比小 batch 大 5000 倍，同一个随机扰动对大 batch 来说微乎其微，对小 batch 却是毁灭性打击。 结果：小 batch 看起来 sharp（假象），大 batch 看起来 flat（假象）。 和真实情况完全相反。
为什么大 batch 权重这么大？ 没有 weight decay 时，大 batch 的梯度噪声小，每次更新方向一致，权重持续增长（类似随机游走的漂移）。 小 batch 的梯度噪声大，方向来回摆动，权重被约束在较小范围。

Figure 3: 用 filter normalization 之后的 1D + 2D 可视化

子图说明

子图条件测试误差损失面形状
(a) WD=0, bs=128 7.37% flat（谷底宽）

✅ 归一化之后，pattern 正确了： 小 batch 找到 flat 极小值（泛化好），大 batch 找到 sharp 极小值（泛化差）。 而且加上 weight decay 后，两者差距更明显——flat 的泛化更好（6.00% vs 7.37%）。

怎么看 2D 等高线图

每张 2D 图是一个 f(α, β) = L(θ* + α·δ + β·η) 的等高线图：

  • 中心 (0,0) = 训练得到的参数 θ*
  • δ, η = 两个经过 filter normalization 的随机方向
  • 颜色：蓝色/冷色 = loss 低，红色/暖色 = loss 高
  • 绿线 = 测试精度标记

看 (a) 和 (b) 的对比：

(a) bs=128： 从中心往四周走，loss 缓慢上升（蓝色区域大）→ flat
(b) bs=8192： 从中心往四周走，loss 急剧上升（蓝色区域小，很快变红）→ sharp

(c) bs=128+WD： 蓝色区域最大，loss 面最平坦 → 泛化最好 (6.00%)
(d) bs=8192+WD： 蓝色区域小，sharp → 泛化差 (10.19%)

Figure 2 → Figure 3 的故事线

  1. Figure 2（没归一化）： 画出来的 loss 图是误导的。小 batch 看起来 sharp，大 batch 看起来 flat——完全反了。原因是大 batch 的权重比小 batch 大 5000 倍。
  2. Figure 3（filter normalization）： 把扰动缩放到和权重同尺度后，正确的图案出现了——小 batch 确实 flat，大 batch 确实 sharp。
  3. 这就是 filter normalization 的必要性证明。 没有它，单张图本身有意义（显示沿某方向 loss 变化），但不同图之间不能比较 sharpness。
