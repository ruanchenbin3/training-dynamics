1. 动机：三个矛盾

矛盾 1：理论说不可能，实际上很好训

理论上神经网络损失面是高维非凸的，应该有无数鞍点和糟糕的局部极小值， SGD 不应该收敛。但实践中每次 SGD 都收敛到零训练损失，而且泛化得很好。

$$ \min_{\theta} \mathcal{L}(\theta) = \frac{1}{m}\sum_{i=1}^m \ell(f(x_i; \theta), y_i) $$

矛盾 2：Sharp vs Flat 极小值之争

小 batch SGD 找到 flat 极小值（泛化好），大 batch 找到 sharp 极小值（泛化差）， 这是 Keskar et al. 2017 的结论。但 Dinh et al. 2017 指出测量的 sharpness 受尺度不变性影响， 可能是假象。

矛盾 3：跳过连接为什么有效

ResNet 能训到 1000+ 层，普通 CNN 20 层就开始崩。 简单的"梯度更好传"解释不够——它对损失面的几何做了什么？

2. 问题：无法比较的损失面

神经网络的参数空间是百万维的，可视化必须降到 2D：

$$ f(\alpha, \beta) = \mathcal{L}(\theta^* + \alpha \delta + \beta \eta) $$

其中 $\delta, \eta$ 是两个随机方向向量。

尺度不变性破坏了一切

由于 ReLU + Batch Normalization，网络有 尺度不变性 （scale invariance）：

$$ f(x; \theta) = f(x; \lambda \theta) \quad \text{(ReLU + BN)} $$

即两个参数大小差 10 倍的网络可能是数学等价的。 但画损失面时，一个固定的随机扰动 $\delta$ 对大权重网络影响小（看起来"平滑"）， 对小权重网络影响大（看起来"尖锐"）——这完全是测量假象。

Dinh et al. 的论点： 你看到的 sharp/flat 差异可能只是尺度不变性的产物。 没有适当的归一化，任何 sharpness 比较都是不可靠的。

3. 解决方案：Filter Normalization

核心思想：把随机扰动方向 $\delta$ 缩放到和网络权重 $\theta$ 同尺度。

数学定义

令 $\theta$ 为网络参数，$\delta$ 为随机高斯方向向量。 对第 $i$ 层的第 $j$ 个 filter：

$$ \delta_{i,j} \leftarrow \delta_{i,j} \cdot \frac{\|\theta_{i,j}\|}{\|\delta_{i,j}\|} $$

其中 $\|\cdot\|$ 是 Frobenius 范数：

$$ \|W\|_F = \sqrt{\sum_{i=1}^m \sum_{j=1}^n w_{ij}^2} $$

为什么是 filter-wise 而不是 layer-wise

同一层内不同 filter 的大小可能不同（边缘检测 filter 和纹理检测 filter 的参数幅度不同）， layer-wise 归一化会抹平这种差异。Filter-wise 归一化尊重每个 filter 的内部结构：

归一化方式做法问题
无归一化$\delta$ 直接用高斯随机尺度不同，不可比
Layer-wise整层用同一个缩放因子层内 filter 差异被抹平
Filter-wise ✓每个 filter 独立缩放正确保留内部结构

适用于所有层

卷积层：filter = 卷积核。
全连接层：等价于 $1 \times 1$ 卷积，每个神经元对应一个 filter。

验证

关键结果： 使用 filter normalization 后，sharpness 与泛化误差强相关 （$R^2 > 0.9$）。不归一化则是噪声，甚至可能负相关。

4. 核心发现

发现 1：跳过连接抑制混沌相变

Figure 1 (Li et al., Fig 1 的复现)： 左侧 Plain Net（无跳过连接）损失面混沌；右侧 ResNet（有跳过连接）光滑

当网络深度超过某个阈值时，Plain Net 的损失面从"近似凸"突然转变为"混沌" （chaotic），伴随测试精度骤降，最终无法训练。跳过连接抑制了这个相变：

$$ \text{Plain Net: } \quad \text{depth} > 20 \implies \text{chaotic} $$ $$ \text{ResNet: } \quad \text{depth} < 1000 \implies \text{smooth} $$

发现 2：Sharp 极小值泛化更差（用正确方法测量后）

使用 filter normalization 后，sharpness 与泛化误差正相关：

$$ \text{test error} \propto \lambda_{\max}(\nabla^2 \mathcal{L}(\theta^*)) $$

其中 $\lambda_{\max}$ 是 Hessian 的最大特征值。 Hessian 的负特征值（损失面中的"沟壑"）数量也与泛化误差正相关。

发现 3：SGD 轨迹在低维空间

虽然参数空间是百万维，SGD 的实际轨迹只用了约 5 个有效维度。 这解释了为什么 2D 投影能捕捉到真实结构——SGD 本质上在低维子空间里游走。

5. 对实践的启示

场景启示
设计生成模型（U-Net）跳过连接不只是梯度通道——它让损失面几何更友好
比较不同训练方法用 filter normalization 画 loss landscape，客观比较
泛化差量 Hessian 最大特征值，sharp 则考虑 SWA 平滑
深层网络训不动检查是否出现了 chaotic transition，加跳过连接
多 loss 加权可视化每个 loss 的梯度方向一致性，确认是否冲突
关于你的 T1 mapping 项目： 你的多输入 U-Net（5 contrast → T1 map）天然享受了跳过连接带来的光滑损失面。 如果后续加深网络或加对抗 loss 后训练不稳定，可以考虑检查 loss landscape 是否出现了混沌。

附录：Q&A 记录

Q2: 为什么是 filter-wise 而不是 layer-wise？

同一个卷积层内，不同 filter 的参数规模可能差几十倍： 

# Filter 1: 边缘检测器（权重很大）
||filter_1|| = 3.0

# Filter 2: 纹理检测器（权重很小）
||filter_2|| = 0.05

Layer-wise 归一化的问题： 整层用一个缩放因子 $||\theta_{\text{layer}}|| \approx ||\theta_1|| = 3.0$这么大的尺子去量 filter_2（0.05）， 相当于在微小参数上给了 60 倍的扰动——损失面会在 filter_2 方向看起来"sharp"， 但这是尺子问题，不是真正的几何。

Filter-wise 归一化： 每个 filter 用自己独立的尺度，各量各的——扰动比例都是 100%，公平可比。

详见论文 Section 4: Proposed Visualization — Filter-Wise Normalization

Q4: filter 是什么意思，和卷积核什么关系？

论文中的 filter = 卷积核 = kernel，是同一个概念。

一个卷积层：输入 64×64×3，用 32 个 filter（每个 3×3×3）→ 输出 64×64×32
每个 filter 检测一种模式（边缘、纹理、颜色等），输出一个通道。

全连接层也适用：每个神经元等价于一个 1×1 的 filter。

所以 filter-wise 归一化 = 每个卷积核单独算范数、独立缩放。

Q3: 举个具体的卷积网络数值例子

一个卷积层，2个 filter，每个 2×2，权重差 100 倍：

filter_1: [[ 1.0, -1.0], [ -1.0, 1.0]]     → ||θ₁|| = 2.0
filter_2: [[ 0.01, -0.01], [-0.01, 0.01]]  → ||θ₂|| = 0.02

Layer-wise： 整层范数 ≈ 2.0，被 filter_1 主导。缩放因子 ≈ 1.4。 对 filter_2（范数 0.02）来说，1.4 的扰动是 72 倍 过度扰动。 画出来 filter_2 方向 Loss 暴跳，看着 sharp——但这是尺子太大，不是真 sharp。

Filter-wise： filter_1 用 2.01 倍缩放，filter_2 用 0.02 倍缩放。 两个 filter 的扰动比例都是精确的 100%。 画出来的 sharpness 反映真实几何。

结论：Layer-wise 用大 filter 的尺子量小 filter 的鞋，filter-wise 各用各的尺子。

Q1: 小 batch → flat, 大 batch → sharp 是什么意思？

直观理解：

Flat 极小值：                      Sharp 极小值：

    \      /                        \            /
     \    /                          \          /
      \  /                            \        /
       \/                              \______/

训练loss≈0，谷底宽                   训练loss≈0，谷底窄
对测试偏移不敏感                     对测试偏移极其敏感

数值例子： 两个模型训练 loss 都是 0，但测试集最优解从 $\theta^*=1$ 偏到了 $\theta=1.05$：

$$ \text{Flat: } \mathcal{L}(1.05) \approx 0.0105 \quad\text{(几乎没变)} $$ $$ \text{Sharp: } \mathcal{L}(1.05) = 0.25 \quad\text{(暴跳 25 倍)} $$

为什么 batch size 影响： 小 batch 的梯度噪声大，参数不断"抖动"，只能在宽谷底站稳（尖谷底一抖就掉出去）。 大 batch 梯度精确，直接冲入最深的谷底——但那个谷底可能是 sharp 的。
